Colori & Fonts

1. Colori

I colori possono essere dichiarati in diversi modi

                
.default {
    color: green;
}
.esadecimale {
    color: #ff0000;
}
.rgb {
    color: rgb(218, 204, 15);
}
.rgba {
    background-color: rgba(115, 151, 60, 0.6);
}
.hsla {
    background-color: hsla(214, 83%, 37%, 0.3);
}
  1. <li class="default" > colore con valore default </li>
  2. <li class="esadecimale" > colore con valore esadecimale </li>
  3. <li class="rgb" > colore con valore rgb </li>
  4. <li class="rgba" > colore con valore rgba </li>
  5. <li class="hsla" > colore con valore hsla </li>

2. Fonts

                
.graziato {
    font-family: Times, Times New Roman, Georgia, serif;
}
.bastoni {
    font-family: arial, sans-serif;
}

<p class="graziato" >font graziato</p>

<p class="bastoni" >font bastoni</p>

NB serve dare più opzioni in caso l'utente non abbia installata la prima scelta, o, in alternativa dichiarare un @font-face

esempio:

                
@font-face {
    font-family: "Open Sans";
    src: url("/fonts/OpenSans-Regular-webfont.woff2") format("woff2"),
            url("/fonts/OpenSans-Regular-webfont.woff") format("woff");
}

NBB quando importo un google font, sto importando un css che ha dichiarato il font face hostato in cdn google

può essere un valore perché ci semplifica il lavoro, può essere un problema se i nostri utenti lavorano sotto una rete con delle limitazioni.

                
<style>
    @import url('https://fonts.googleapis.com/css2?family=Roboto&display=swap');
    .roboto {font-family: 'Roboto', sans-serif;}
</style>

<p class="roboto" >font roboto</p>
